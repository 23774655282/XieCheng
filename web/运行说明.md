# 易宿酒店预订平台 - 前端运行说明

## 一、环境要求

| 依赖 | 版本要求 |
|------|----------|
| Node.js | 18 及以上 |
| npm | 随 Node.js 安装 |

## 二、安装步骤

### 2.1 进入项目目录

```bash
cd web
```

### 2.2 安装依赖

```bash
npm install
```

## 三、运行命令

### 3.1 开发模式（推荐）

启动 Vite 开发服务器，支持热更新：

```bash
npm run dev
```

启动后，在浏览器访问终端提示的地址（通常为 `http://localhost:5173`）。

### 3.2 构建生产版本

```bash
npm run build
```

构建产物输出到 `dist` 目录，可部署到任意静态托管服务。

### 3.3 预览生产构建

本地预览构建后的效果：

```bash
npm run preview
```

### 3.4 代码检查

```bash
npm run lint
```

## 四、项目结构说明

- **用户端**：首页、酒店列表、酒店详情（移动优先，响应式）
- **管理端**：登录、注册、商户酒店管理、管理员审核

## 五、主要路由

| 页面 | 路径 |
|------|------|
| 用户首页 | `/` |
| 酒店列表 | `/hotels` |
| 酒店详情 | `/hotels/:id` |
| 管理端登录 | `/admin/login` |
| 管理端注册 | `/admin/register` |
| 商户 - 我的酒店 | `/admin/merchant/hotels` |
| 商户 - 新增酒店 | `/admin/merchant/hotels/new` |
| 商户 - 编辑酒店 | `/admin/merchant/hotels/:id/edit` |
| 管理员 - 酒店审核 | `/admin/audit/hotels` |

## 六、与后端联调

前端需要对接后端 API 时，请确保：

1. **后端服务已启动**：默认运行在 `http://localhost:3000`
2. **跨域配置**：后端已启用 CORS，支持前端开发服务器跨域请求
3. **API 地址**：若后端端口或域名不同，需在前端配置中修改 API 基础地址（如使用环境变量 `VITE_API_BASE_URL`）

## 七、常见问题

**Q：开发服务器端口被占用？**  
A：Vite 会自动尝试下一个可用端口，或可在 `vite.config.ts` 中通过 `server.port` 指定端口。

**Q：构建后部署到子路径？**  
A：在 `vite.config.ts` 中设置 `base: '/你的子路径/'`。

**Q：如何配置 API 代理？**  
A：在 `vite.config.ts` 的 `server.proxy` 中配置，例如将 `/api` 代理到 `http://localhost:3000`，可避免跨域问题。
